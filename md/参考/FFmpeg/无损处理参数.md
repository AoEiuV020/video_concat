# FFmpeg 无损处理参数参考

本文档整理 FFmpeg 在 stream copy（`-c copy`）模式下可用的无损处理参数，所有操作不涉及重编码。

## 基础命令结构

```bash
ffmpeg -y -f concat -safe 0 -i filelist.txt [额外参数...] output.mp4
```

| 参数 | 说明 |
|------|------|
| `-y` | 自动覆盖输出文件 |
| `-f concat -safe 0 -i filelist.txt` | 基于文件列表的[拼接模式](https://ffmpeg.org/ffmpeg-formats.html#concat-1) |
| `-c copy` 或 `-vcodec copy -acodec copy` | stream copy，不重编码 |

## 去除音频

```bash
ffmpeg -i input.mp4 -c copy -an output.mp4
```

| 参数 | 说明 |
|------|------|
| `-an` | 禁用所有[音频输出流](https://ffmpeg.org/ffmpeg.html#Audio-Options) |

使用 `-an` 时不能同时指定 `-acodec copy`，两者互斥。

## 去除字幕

```bash
ffmpeg -i input.mp4 -c copy -sn output.mp4
```

| 参数 | 说明 |
|------|------|
| `-sn` | 禁用所有[字幕输出流](https://ffmpeg.org/ffmpeg.html#Subtitle-options) |

## 元数据旋转

通过 display matrix 标记旋转角度，不改变像素数据，播放器根据标记旋转显示。

```bash
ffmpeg -display_rotation:v:0 90 -i input.mp4 -c copy output.mp4
```

| 参数 | 说明 |
|------|------|
| `-display_rotation:v:0 X` | 设置第一条视频流的旋转角度（逆时针），见[官方文档](https://ffmpeg.org/ffmpeg.html#Advanced-Video-options)。X 为角度（0/90/180/270） |

> **注意**：`-display_rotation` 是**输入选项**，必须放在 `-i` 之前。

### 已废弃的旋转方式

| 写法 | 状态 | 说明 |
|------|------|------|
| `-metadata:s:v rotate=90` | ❌ 已废弃 | 不带流索引，[不生效](https://ffmpeg.org/pipermail/ffmpeg-user/2024-July/058428.html) |
| `-metadata:s:v:0 rotate=90` | ❌ 已废弃 | 带流索引，现代 FFmpeg (7.0+) 下同样不生效 |

FFmpeg 邮件列表 [2024-07](https://ffmpeg.org/pipermail/ffmpeg-user/2024-July/058428.html) 明确说 `-metadata:s:v rotate` "deprecated and does not work"，并推荐 `-display_rotation`。

### 版本要求

`-display_rotation` 于 [2022-10-19](https://github.com/FFmpeg/FFmpeg/commit/c889248647780753ec8b05138c7de4a707adb106) 加入 FFmpeg，首次包含在 FFmpeg 6.0 中。**本项目要求 FFmpeg ≥ 6.0**。

> 仅修改播放器显示方向，实际像素不变。不尊重 display matrix 的播放器（如旧版 Windows Media Player）仍会显示原始方向。

## 快速启动（Fast Start）

将 MP4 的 moov atom 移至文件开头，优化网络流播放：

```bash
ffmpeg -i input.mp4 -c copy -movflags +faststart output.mp4
```

| 参数 | 说明 |
|------|------|
| [`-movflags +faststart`](https://ffmpeg.org/ffmpeg-formats.html#Options-11) | moov atom 前置，仅对 MP4/MOV 容器有效 |

其他容器格式（MKV/AVI/WebM）设置此参数无效果，不会报错但会被忽略。

## 清除元数据

移除容器级别的所有元数据（拍摄设备、GPS、标题等）：

```bash
ffmpeg -i input.mp4 -c copy -map_metadata -1 output.mp4
```

| 参数 | 说明 |
|------|------|
| [`-map_metadata -1`](https://ffmpeg.org/ffmpeg.html#Advanced-options) | 不从任何输入映射元数据到输出 |

> 注意：此选项清除的是容器级元数据。流级别的 codec 信息（如分辨率、编码格式）不受影响。

## 章节注入

使用 [FFMETADATA1 格式](https://ffmpeg.org/ffmpeg-formats.html#Metadata-1)注入章节标记：

### 元数据文件格式

```
;FFMETADATA1
[CHAPTER]
TIMEBASE=1/1000
START=0
END=149999
title=Part 1

[CHAPTER]
TIMEBASE=1/1000
START=150000
END=299999
title=Part 2
```

| 字段 | 说明 |
|------|------|
| `TIMEBASE=1/1000` | 时间单位为毫秒 |
| `START` / `END` | 章节起止时间（毫秒） |
| `title` | 章节标题 |

### 注入命令

```bash
ffmpeg -i input.mp4 -i chapters.txt -map_metadata 1 -c copy output.mp4
```

| 参数 | 说明 |
|------|------|
| `-i chapters.txt` | [元数据文件](https://ffmpeg.org/ffmpeg-formats.html#Metadata-1)作为第二输入 |
| `-map_metadata 1` | 从输入索引 1（章节文件）[映射元数据](https://ffmpeg.org/ffmpeg.html#Advanced-options)到输出 |

章节注入适用于 MP4、MKV 等[支持章节的容器](https://ikyle.me/blog/2020/add-mp4-chapters-ffmpeg)。AVI 和 WebM 对章节支持有限。

## 参数组合注意事项

| 场景 | 注意 |
|------|------|
| `-an` + `-acodec copy` | 互斥，不能同时使用 |
| `-map_metadata -1` + 章节注入 | 章节通过 `-map_metadata 1` 引入，与 `-map_metadata -1` 冲突；需二选一 |
| `-metadata:s:v:0 rotate=X` + `-map_metadata -1` | `-map_metadata -1` 会清除容器元数据，但 `-metadata:s:v:0` 是流级别，两者不冲突 |
| `-movflags +faststart` + 非 mp4/mov | 参数被忽略，不报错 |
