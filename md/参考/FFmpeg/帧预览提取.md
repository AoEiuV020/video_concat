# 帧预览提取

从视频中提取单帧画面用于预览显示。本文档聚焦 `-c copy` 无损工作流中的预览需求，提取操作本身涉及解码但不影响最终输出。

## 基础命令

```bash
ffmpeg -ss <time> -i input.mp4 -vframes 1 output.jpg
```

| 参数 | 说明 |
|------|------|
| [`-ss`](https://ffmpeg.org/ffmpeg.html#Main-options) | 寻址到指定时间（放在 `-i` 前为输入级寻址） |
| `-i` | 输入文件 |
| [`-vframes 1`](https://ffmpeg.org/ffmpeg.html#Video-Options) | 只输出 1 帧视频 |
| `output.jpg` | 输出文件，格式由扩展名决定 |

## 关键帧预览 vs 任意帧预览

| 对比项 | 关键帧预览 | 任意帧预览 |
|--------|-----------|-----------|
| `-ss` 位置 | `-i` 之前（输入级） | `-i` 之后（输出级） |
| 寻址行为 | 直接跳转到目标关键帧 | 需从前一个关键帧解码到目标帧 |
| 速度 | **极快**（~50ms），直接定位 | 较慢，取决于离前一个关键帧的距离 |
| 解码量 | 仅 1 帧（I帧自包含） | 需解码从前一个关键帧到目标帧的所有帧 |
| 画面准确性 | 100% 准确 | 100% 准确 |
| 适用场景 | 本项目（裁剪点只能是关键帧） | 需要帧精确预览的场景 |

**本项目只需要关键帧预览**，因为裁剪点只能选在关键帧上。这意味着：

- `-ss` 始终放在 `-i` 之前，走最快路径
- 不需要解码中间帧，每次提取都是 O(1)
- 拖动进度条时可以实时刷新预览，用户体验流畅

## 输出格式

| 格式 | 命令 | 特点 |
|------|------|------|
| JPEG | `-vframes 1 out.jpg` | 文件小，有损压缩，适合预览 |
| PNG | `-vframes 1 out.png` | 无损，文件较大 |
| BMP | `-vframes 1 out.bmp` | 无压缩，最快但文件最大 |
| 管道输出 | `-f image2pipe -vcodec mjpeg pipe:1` | 不写磁盘，直接输出到 stdout |

### 管道输出（推荐）

避免临时文件，直接将图像数据读入内存：

```bash
ffmpeg -ss 4.004 -i input.mp4 -vframes 1 -f image2pipe -vcodec mjpeg pipe:1
```

| 参数 | 说明 |
|------|------|
| [`-f image2pipe`](https://ffmpeg.org/ffmpeg-formats.html#image2pipe) | 图像管道输出格式 |
| `-vcodec mjpeg` | 编码为 JPEG 格式 |
| `pipe:1` | 输出到 stdout |

在 Dart 中可通过 `Process.run()` 的 stdout 直接获取字节数据，再用 `Image.memory()` 显示。

### 控制输出质量

```bash
# JPEG 质量（2~31，越小质量越高，默认约 2~5）
ffmpeg -ss 4.004 -i input.mp4 -vframes 1 -q:v 5 output.jpg

# 缩放尺寸（降低分辨率加速）
ffmpeg -ss 4.004 -i input.mp4 -vframes 1 -vf scale=320:-1 output.jpg
```

| 参数 | 说明 |
|------|------|
| [`-q:v`](https://ffmpeg.org/ffmpeg.html#Main-options) | 视频质量（JPEG：2=最高，31=最低） |
| [`-vf scale=W:-1`](https://ffmpeg.org/ffmpeg-filters.html#scale) | 缩放到宽度 W，高度按比例自动计算 |

预览用途推荐 `-q:v 5 -vf scale=640:-1`，兼顾速度和清晰度。

## 完整示例

### 提取关键帧预览（本项目场景）

```bash
# 已知关键帧时间为 4.004s，提取预览
ffmpeg -ss 4.004 -i input.mp4 \
  -vframes 1 -q:v 5 -vf scale=640:-1 \
  -f image2pipe -vcodec mjpeg pipe:1
```

### 批量提取（生成缩略图时间线）

```bash
# 每 10 秒提取一帧
ffmpeg -i input.mp4 -vf "fps=1/10,scale=160:-1" thumb_%04d.jpg
```

注意：此方案需要解码全部帧，性能差。如果只需要关键帧缩略图，应改用逐个 `-ss` 提取。

## 性能参考

| 场景 | 耗时（估算） | 说明 |
|------|------------|------|
| 关键帧提取（`-ss` 在 `-i` 前） | ~50ms | 直接定位，无需解码中间帧 |
| 任意帧提取（`-ss` 在 `-i` 后） | ~200ms~2s | 取决于与前一个关键帧的距离 |
| 关键帧提取 + 缩放到 640px | ~80ms | 增加少量缩放开销 |
| 管道输出 vs 文件输出 | 差异微小 | 管道省去磁盘 I/O |

## 注意事项

| 事项 | 说明 |
|------|------|
| `-ss` 精度 | 输入级寻址会跳到最近的关键帧；本项目传入的就是关键帧时间，无精度问题 |
| 并发提取 | 可并行运行多个 ffmpeg 进程提取不同时间点的预览 |
| 缓存策略 | 已提取的预览图可缓存（以文件路径+时间戳为 key），避免重复提取 |
| 错误处理 | 时间超出视频范围时 ffmpeg 会输出最后一帧或报错，需捕获 |
| `-nostdin` | 批量/自动化调用时加 [`-nostdin`](https://ffmpeg.org/ffmpeg.html#Main-options) 防止 ffmpeg 等待 stdin 输入 |
