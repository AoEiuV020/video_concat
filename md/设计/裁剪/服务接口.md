# 服务接口

裁剪功能依赖两类能力：**关键帧探测**与**连续视频预览**。前者负责确定可裁剪时间点，后者负责让滑块、关键帧和视频画面保持一致。

## 关键帧探测

### 局部探测

在指定时间窗口内查找关键帧，不遍历全文件。

| 参数 | 说明 |
|------|------|
| filePath | 视频文件路径 |
| startTime | 查询窗口起点（微秒） |
| endTime | 查询窗口终点（微秒） |
| 返回 | 关键帧列表，每项包含 PTS（微秒）和 DTS（微秒，可为 null） |

底层命令（时间参数需格式化为秒字符串）：

```bash
ffprobe -read_intervals {startTime}%{endTime} \
  -select_streams v:0 -skip_frame nokey \
  -show_entries frame=pts_time,dts_time \
  -of csv=p=0 input.mp4
```

### 全量探测

获取视频中所有关键帧。适用于需要展示完整时间线的场景。

| 参数 | 说明 |
|------|------|
| filePath | 视频文件路径 |
| 返回 | 关键帧列表，每项包含 PTS（微秒）和 DTS（微秒，可为 null） |

底层命令：

```bash
ffprobe -select_streams v:0 -skip_frame nokey \
  -show_entries frame=pts_time,dts_time \
  -of csv=p=0 input.mp4
```

## 连续视频预览

播放器层必须提供“打开媒体、连续播放、暂停、seek、状态流”这组能力，裁剪页据此实现实时预览与两级 loading 状态。

### 基础能力

| 能力 | 说明 |
|------|------|
| 打开媒体 | 进入裁剪页时以暂停态载入视频 |
| seek 到时间点 | 拖动、prev/next、暂停后吸附都依赖 seek |
| 播放 / 暂停 | 用于连续预览与暂停后关键帧吸附 |
| 获取当前位置 | 驱动进度条和当前时间文字 |
| 完成事件 | 用于识别“播放到末尾” |
| 错误事件 | 用于暴露播放器失败现场 |

### 状态流要求

```mermaid
%%| width: 60%
flowchart LR
    A["用户目标时间"] --> B["播放器 seek"]
    B --> C["当前位置流持续更新"]
    C --> D{"已到达目标?"}
    D -->|否| E["保留预览遮罩"]
    D -->|是| F["清除预览遮罩"]
```

| 事件 / 状态 | 用途 |
|------|------|
| 当前位置流 | 播放时推动进度条，seek 后判断画面是否追上目标 |
| 播放状态流 | 区分播放中与暂停态，决定关键帧相关按钮是否可用 |
| 完成事件 | 将当前时间置为虚拟末尾 |
| 错误事件 | 显示错误信息并打日志 |

### 设计要求

| 规则 | 说明 |
|------|------|
| 时间未吸附 | 播放中、拖动中、关键帧解析中均视为时间不可用 |
| 画面未同步 | 关键帧目标已定，但播放器 position 还没追上目标 |
| seek 成功判定 | 通过当前位置到达目标（允许少量容差）来收口遮罩 |
| 新请求覆盖旧请求 | 新的拖动或导航必须取消前一次等待中的画面同步 |

## 文件列表生成

现有的文件列表生成逻辑需要支持为每个条目追加 inpoint/outpoint 指令。

| 场景 | 生成内容 |
|------|---------|
| 无 trimConfig 或空片段 | 单条 `file 'path'` |
| 有片段 | 每个片段一条 `file 'path'` + `inpoint` + `outpoint` |
| inpoint = 0 | 省略 `inpoint` 行 |
| outpoint = 视频末尾 | 省略 `outpoint` 行 |

### 伪代码

```text
function buildFileList(items):
  for item in items:
    if item.trimConfig 为空或无片段:
      输出 "file '{item.filePath}'"
    else:
      for segment in item.trimConfig.segments:
        输出 "file '{item.filePath}'"
        if segment.inpoint > 0:
          输出 "inpoint {segment.inpoint}"
        if segment.outpoint < item.duration:
          输出 "outpoint {segment.effectiveOutpoint}"
```

effectiveOutpoint 优先使用 DTS，回退到 PTS（见[数据模型](数据模型.md#effectiveoutpoint)）。

## 视频时长获取

裁剪页面需要视频总时长来设置进度条范围。

| 参数 | 说明 |
|------|------|
| filePath | 视频文件路径 |
| 返回 | 时长（微秒） |

可复用现有探测接口的 format.duration 字段，或单独查询：

```bash
ffprobe -select_streams v:0 \
  -show_entries format=duration \
  -of csv=p=0 input.mp4
```
