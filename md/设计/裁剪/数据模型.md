# 数据模型

## 时间戳精度

所有时间戳使用**整数微秒**（`int64`，单位 µs），与 FFmpeg 内部 `av_parse_time` 精度一致。

| 属性 | 说明 |
|------|------|
| 单位 | 微秒（µs），1 秒 = 1,000,000 µs |
| 类型 | 64 位有符号整数 |
| 来源 | ffprobe 输出的浮点字符串 × 1,000,000 后四舍五入 |
| 输出 | 传给 FFmpeg 时格式化为秒字符串（如 `4004000` → `"4.004000"`） |

转换公式：

```
字符串 → 微秒: round(parseDouble(str) * 1000000)
微秒 → 字符串: "${us ~/ 1000000}.${(us % 1000000).toString().padLeft(6, '0')}"
```

## 裁剪片段

一个裁剪片段由一对 inpoint/outpoint 定义，表示从视频中截取的一个连续区间。

| 字段 | 类型 | 说明 |
|------|------|------|
| inpoint | int64（微秒） | 片段起点，已对齐到关键帧 |
| outpoint | int64（微秒） | 片段终点，已对齐到关键帧 |

约束：`outpoint > inpoint`，两者均为关键帧时间戳。

## 裁剪配置

每个视频可配置零到多个裁剪片段。

| 字段 | 类型 | 说明 |
|------|------|------|
| segments | 裁剪片段列表 | 按 inpoint 升序排列，片段间不重叠 |

空列表或 null 表示不裁剪，使用完整视频。

## 视频项扩展

在原有视频项模型基础上增加：

| 新增字段 | 类型 | 说明 |
|----------|------|------|
| trimConfig | 裁剪配置（可选） | null 表示不裁剪 |
| duration | int64（微秒，可选） | 视频总时长，由探测工具获取 |

## 文件列表生成

裁剪配置直接映射到 concat demuxer 的文件列表。每个片段生成一个带 inpoint/outpoint 的条目，同一视频的多个片段产生多条文件引用。

### 映射规则

| 场景 | 文件列表内容 |
|------|-------------|
| 无裁剪 | `file 'path'` |
| 单片段 | `file 'path'` + `inpoint X` + `outpoint Y` |
| 多片段 | 同一 `file 'path'` 出现多次，各自带不同的 inpoint/outpoint |
| inpoint = 0 | 可省略 `inpoint`（从头开始） |
| outpoint = 末尾 | 可省略 `outpoint`（到文件末尾） |

### 示例

三个视频：A 不裁剪，B 取中间一段，C 取两段：

```
file 'A.mp4'

file 'B.mp4'
inpoint 4.004
outpoint 28.028

file 'C.mp4'
outpoint 15.015

file 'C.mp4'
inpoint 60.060
outpoint 90.090
```

## 章节时长计算

启用章节注入时，每个条目的时长需根据裁剪配置重新计算：

| 场景 | 时长计算 |
|------|---------|
| 无裁剪 | 原始视频时长 |
| 单片段 | `outpoint - inpoint` |
| 多片段 | 每个片段各自计算 `outpoint - inpoint`，各为独立章节 |

多片段场景下，同一视频可能产生多个章节。
